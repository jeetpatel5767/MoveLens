01 · Principles

Six commitments
we audit ourselves against.

A vulnerability report can't waver. Neither can the surface that delivers it. These six rules pre-empt every styling decision below — when a screen feels off, walk it through this list. The answer is almost always here.

A · Confidence, not noise
Severity carries meaning.

Red means critical. Amber means look. Green means cleared. No color appears on a finding card unless it changes how an engineer should respond — decoration is a bug.

B · One mascot, one voice
Lens shows up everywhere.

Same plush body, same blue, same goggles — from hero to footer watermark. The character is the product's face, never a sticker. Personality earns trust faster than a logotype can.

C · Built for the chain
Sui-native, on purpose.

Lavender pills, aurora glow, the Sui blue — visual callbacks to the stack we live on. We aren't blending in. We're saying out loud which chain we believe in.

D · Dense for developers
Mono where it matters.

Package addresses, blob IDs, rule_ids, line ranges — always monospace, always copyable, always tabular. The audit report is a document of record.

E · Honest about limits
The disclaimer is designed.

"Automated pre-screen, not a substitute for human audit" gets a designated component, a sticky placement, and a confidence score next to every finding. Trust comes from saying what we are not.

F · Dark by default, never flat
Pure black, broken alive.

Grain, hairline borders, the occasional aurora — black is never a dead void. Depth comes from texture and edge, not from gradient soup. The screen breathes.

02 · Typography

Three voices,
one steady tone.

Cabinet Grotesk carries the weight — the marketing punch, the report titles. Switzer runs the working surface: body, UI, labels. IBM Plex Mono is the voice of record — every address, every blob ID, every rule_id we ever ship.

Cabinet Grotesk
Display · Titles · Marketing

Weights 700–800. Tight tracking (−0.03em). Reserved for ≥28px. We don't use it for body — the wide aperture wants air.

Audit before deploy.
Switzer
UI · Body · Labels

Weights 400 / 500 / 600. Neutral grotesque, calm at long lengths. Default body 14px / 1.55. Never above 600 — hierarchy comes from scale and color.

Lens read your package, flagged four issues across sources/vault.move, and signed the report with the same key it published from. You can verify any of it on-chain.
IBM Plex Mono
IDs · Code · Voice of record

Addresses, blob IDs, rule_ids, line numbers, code excerpts. Tabular figures. Uppercase wide-tracked variant carries small labels and section codes.

04 · Space · Radius · Elevation

4-unit grid,
soft corners, faint lift.

Spacing is on a 4px rhythm — tight enough for dense audit tables, generous enough that a finding card never feels cramped. Radii are intentionally round (16–24px) to echo the mascot's softness. Dark-mode elevation is a whisper: a hairline border, a faint glow on hover. No floating cards.

05 · Mascot

Meet Lens.
Plush, blue, watching.

Lens is a chunky soft-bodied bug character cast in a single matte Sui-blue plush. The one signature accessory is a pair of oversized scanner goggles — Lens reads code for a living, and the goggles say so. Three contexts: hero (full body, confident), report icon (small, beside the disclaimer), footer watermark (huge, low opacity). Same proportions everywhere. No new poses without a reason.

✓ Do

✗ Don't

✗ Don't
Recolor the body for severity. Lens is always blue. Severity lives on the badge, not the mascot.

08 · Trust panel

Aurora,
earned sparingly.

The aurora panel is the single place we let color speak for itself. It marks the moment a report becomes a trust artifact: signed, encrypted, stored, and linked to its package on-chain. Two per page maximum.

Signed · encrypted · stored · linked

Built on the Sui stack we trust.

Every MoveLens report is a four-layer artifact: encrypted by Seal, stored permanently on Walrus, addressed by MVR, anchored to the Sui package's on-chain identity. Anyone can verify the signature — the report can't be tampered with after the fact.
